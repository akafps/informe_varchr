# 07. Recomendaciones y Conclusiones

## Recomendaciones Técnicas y Controles de Seguridad
Para mitigar el riesgo de un incidente similar, se propone la implementación del siguiente esquema de controles:
1. Autenticación de Múltiple Factor (MFA): Obligatorio para todos los accesos remotos y cuentas con privilegios de desarrollador o administrador.
2. Segmentación de Redes: Aislar los servidores que contienen propiedad intelectual (códigos de juegos) de las redes administrativas de uso diario.
3. Respaldos Inmutables (Backups Off-site): Mantener copias de seguridad de las bases de datos desconectadas de la red principal para garantizar la restauración ante ransomware sin pagar rescates.
4. Parches y Actualizaciones: Establecer una política estricta de actualización inmediata de sistemas operativos y VPNs para evitar el uso de vulnerabilidades conocidas.

## Reflexión Final
El caso de Capcom demuestra que las empresas de entretenimiento digital manejan un patrimonio de datos críticos tan valioso como cualquier entidad financiera. La implementación de la Ley N° 21.459 en Chile dota al país de mejores herramientas para perseguir el cibercrimen, pero impone a los desarrolladores y administradores informáticos el desafío de elevar los estándares éticos y técnicos de ciberseguridad para garantizar la confianza de los usuarios.
